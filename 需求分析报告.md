# 合家云社区物业管理平台
## 需求分析报告

---

## 1. 项目背景

### 1.1 项目名称
合家云社区物业管理平台

### 1.2 项目类型
数据库应用系统开发（B/S架构）

### 1.3 开发目的
为社区物业管理部门设计的管理信息系统，实现对社区资产（小区、楼栋、单元、房屋）和业主信息的统一管理，支持多角色权限控制和数据统计，提高物业管理效率。

### 1.4 开发环境
- **操作系统**：Microsoft Windows
- **数据库**：MySQL 8.0
- **后端**：JDK 21 + Spring Boot 3.2.0 + MyBatis-Plus 3.5.5
- **前端**：Vue 3 + Vite 5 + Element Plus 2.x
- **接口技术**：REST API + JDBC

---

## 2. 功能需求

### 2.1 角色划分

系统分为两种角色：

| 角色 | 角色标识 | 权限说明 |
|------|---------|---------|
| 管理员 | admin | 拥有系统所有权限，可进行所有增删改操作 |
| 普通用户 | user | 仅可查看数据，无任何写操作权限 |

默认账号：
- 管理员：`admin` / `123456`
- 普通用户：`operator` / `123456`

### 2.2 功能模块

#### 2.2.1 系统管理模块

| 功能 | 功能说明 | 操作类型 |
|------|---------|---------|
| 用户管理 | 用户的增删改查、角色分配、状态管理 | CRUD |
| 角色管理 | 角色的增删改查 | CRUD |
| 管理员保护 | admin账号不能被其他管理员修改或删除 | 保护 |

#### 2.2.2 社区管理模块

| 功能 | 功能说明 | 操作类型 |
|------|---------|---------|
| 小区管理 | 小区的增删改查 | CRUD |
| 楼栋管理 | 楼栋的增删改查，隶属小区 | CRUD |
| 单元管理 | 单元的增删改查，隶属楼栋 | CRUD |
| 房屋管理 | 房屋的增删改查，隶属单元，状态管理（空置/已入住/装修中） | CRUD |
| 层级约束 | 删除时有下级数据则禁止删除（RESTRICT约束） | 约束 |

层级关系：`小区 → 楼栋 → 单元 → 房屋`

#### 2.2.3 业主管理模块

| 功能 | 功能说明 | 操作类型 |
|------|---------|---------|
| 业主档案 | 业主的增删改查，支持多条件搜索（姓名、手机号） | CRUD |
| 人房绑定 | 业主与房屋的绑定关系管理 | CRUD |
| 关系类型 | 支持业主、家属、租客三种绑定关系 | 约束 |
| 租客唯一 | 同一房屋只能有一个租客 | 约束 |
| 状态联动 | 业主绑定后房屋状态变为"已入住"，解绑后变为"空置" | 联动 |

#### 2.2.4 数据统计模块

| 功能 | 功能说明 |
|------|---------|
| 首页统计 | 显示小区总数、楼栋总数、房屋总数、业主总数 |
| 资产统计 | 各小区下楼栋数、房间数汇总 |
| 房屋状态分布 | 空置数、已入住数、装修中数及占比 |

### 2.3 数据查询需求

各管理模块支持以下查询方式：

| 查询类型 | 说明 |
|---------|------|
| 分页查询 | 默认每页10条，支持20/50条每页 |
| 精确查询 | 按ID精确匹配 |
| 模糊查询 | 按名称等字段模糊搜索 |
| 条件组合 | 支持多条件组合查询 |
| 层级过滤 | 楼栋按小区过滤、单元按楼栋过滤、房屋按单元过滤 |

## 3. 数据需求

### 3.1 数据实体

| 实体 | 说明 | 数量 |
|------|------|------|
| 社区资产 | 小区、楼栋、单元、房屋四级层级结构 | 4张表 |
| 权限管理 | 用户、角色、用户角色关联 | 3张表 |
| 业主管理 | 业主档案、人房绑定关系 | 2张表 |

共 **9张核心数据表**。

### 3.2 数据关系

```
小区（1）──（N）楼栋（1）──（N）单元（1）──（N）房屋（1）──（N）人房绑定（N）──（1）业主
                                                      │
                                                      └─── 关系类型：业主/家属/租客
```

### 3.3 数据完整性约束

| 约束类型 | 实现方式 |
|---------|---------|
| 主键约束 | 每张表 id 为主键自增 |
| 外键约束 | 层级表使用 ON DELETE RESTRICT |
| 唯一约束 | 用户名唯一、角色标识唯一 |
| 非空约束 | 必填字段设置 NOT NULL |
| 状态约束 | 房屋状态只能是 vacant/occupied/decorated |
| 业务约束 | 同一房屋只能有一个租客 |

### 3.4 测试数据

| 数据类型 | 数量 |
|---------|------|
| 小区 | 5个 |
| 楼栋 | 30个 |
| 单元 | 59个 |
| 房屋 | 374间 |
| 业主 | 30人 |
| 人房绑定 | 49条 |

## 4. 非功能需求

### 4.1 性能需求
- 页面响应时间 < 2秒
- 支持至少100个用户同时在线

### 4.2 安全需求
- 普通用户无法访问管理功能
- admin账号受保护，无法被其他用户修改
- 写操作需权限校验

### 4.3 界面需求
- 使用Element Plus组件库，风格统一
- 支持主流浏览器（Chrome、Firefox、Edge）
- 响应式布局
