### 1.2 实体类设计

#### Community（小区）

```java
@TableName("hjy_community")
public class Community {
    private Long id;          // 主键
    private String name;      // 小区名称
    private String address;    // 地址
    private BigDecimal area;  // 占地面积
    private Integer status;   // 状态
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
}
```

#### Building（楼栋）

```java
@TableName("hjy_building")
public class Building {
    private Long id;
    private Long communityId; // 外键 → hjy_community
    private String name;
    private Integer floors;
    private String remark;
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
    @TableField(exist = false)
    private String communityName; // 虚拟字段，存储小区名称
}
```

#### Room（房屋）

```java
@TableName("hjy_room")
public class Room {
    private Long id;
    private Long unitId;      // 外键 → hjy_unit
    private String roomNo;    // 房间号
    private Integer floor;    // 楼层
    private BigDecimal area;  // 面积
    private String roomType;  // 户型
    private String status;     // vacant/occupied/decorated
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
    // 虚拟字段，用于显示上级信息
    @TableField(exist = false)
    private String communityName;
    @TableField(exist = false)
    private String buildingName;
    @TableField(exist = false)
    private String unitName;
}
```

## 3. 关键算法设计

### 3.1 房屋状态联动算法

```
绑定时：
  IF relation_type == 'owner' THEN
      UPDATE room.status = 'occupied'

解绑时：
  IF 房屋无其他有效绑定 THEN
      UPDATE room.status = 'vacant'
```

### 3.2 三级联动过滤算法

```
前端搜索条件联动（以房屋为例）：
  选择小区 → 过滤楼栋下拉 → 清空单元和房屋选择
  选择楼栋 → 过滤单元下拉 → 清空房屋选择
  选择单元 → 过滤完成
```

### 3.3 权限判断算法

```
后端权限判断：
  IF roleKey != 'admin' AND (add/update/delete operation) THEN
      RETURN 403 "无权限操作"

前端界面控制：
  IF roleKey != 'admin' THEN
      隐藏"新增"按钮
      隐藏"编辑"/"删除"按钮
```

## 4. 事务流程

### 4.1 人房绑定事务

```
1. 检查是否已存在相同业主+房屋+有效状态的绑定
   └─ 存在 → 回滚，返回"该业主已绑定此房屋"
2. 如果是租客关系，检查该房屋是否已有租客
   └─ 已有 → 回滚，返回"该房屋已有租客"
3. 插入绑定记录到 hjy_owner_room
4. 如果是业主关系，更新房屋状态为 occupied
5. 提交事务
```

### 4.2 删除房屋事务

```
1. 查询房屋信息
2. 检查房屋状态是否为"已入住"
   └─ 已入住 → 回滚，返回"该房屋已入住，无法删除"
3. 检查是否存在有效绑定
   └─ 有绑定 → 回滚，返回"该房屋仍有绑定关系"
4. 执行 DELETE FROM hjy_room WHERE id = ?
5. 提交事务
```

详细事务设计见《事务设计文档.md》。

## 5. 数据库物理设计

### 5.1 存储引擎
所有表使用 **InnoDB** 引擎，支持事务和外键。

### 5.2 字符集
统一使用 **utf8mb4**，支持emoji和特殊字符。

### 5.3 字段类型选择

| 字段类型 | 使用场景 |
|---------|---------|
| BIGINT | 主键、ID |
| VARCHAR(50/100/255) | 名称、地址等字符串 |
| DECIMAL(10,2) | 面积、金额等数值 |
| DATETIME | 时间戳字段 |
| TINYINT | 状态字段（0/1） |
| DATE | 生日等日期 |

### 5.4 物理存储位置
数据库 `hejiayun`，所有表存储在默认表空间。
