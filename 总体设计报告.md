### 2.2 数据表汇总（共9张）

| 序号 | 表名 | 说明 | 主键 | 外键 |
|------|------|------|------|------|
| 1 | hjy_community | 小区表 | id | - |
| 2 | hjy_building | 楼栋表 | id | community_id → hjy_community |
| 3 | hjy_unit | 单元表 | id | building_id → hjy_building |
| 4 | hjy_room | 房屋表 | id | unit_id → hjy_unit |
| 5 | sys_user | 用户表 | id | - |
| 6 | sys_role | 角色表 | id | - |
| 7 | sys_user_role | 用户角色关联表 | id | user_id → sys_user, role_id → sys_role |
| 8 | hjy_owner | 业主档案表 | id | - |
| 9 | hjy_owner_room | 人房关联表 | id | owner_id → hjy_owner, room_id → hjy_room |

### 2.3 索引设计

| 表名 | 索引类型 | 索引字段 | 说明 |
|------|---------|---------|------|
| sys_user | UNIQUE | username | 用户名唯一 |
| sys_user_role | UNIQUE | (user_id, role_id) | 联合唯一 |
| hjy_owner_room | UNIQUE | (owner_id, room_id, relation_type) | 防止重复绑定 |

## 3. 接口设计

### 3.1 接口规范

- **基础路径**：`/api`
- **请求格式**：JSON
- **响应格式**：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```
- **权限校验**：请求头 `X-Username` 和 `X-User-Role`

### 3.2 接口列表

| 模块 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 登录 | /api/login | POST | 用户登录 |
| 用户管理 | /api/system/user/* | GET/POST/PUT/DELETE | 用户CRUD |
| 角色管理 | /api/system/role/* | GET/POST/PUT/DELETE | 角色CRUD |
| 小区管理 | /api/community/community/* | GET/POST/PUT/DELETE | 小区CRUD |
| 楼栋管理 | /api/community/building/* | GET/POST/PUT/DELETE | 楼栋CRUD |
| 单元管理 | /api/community/unit/* | GET/POST/PUT/DELETE | 单元CRUD |
| 房屋管理 | /api/community/room/* | GET/POST/PUT/DELETE | 房屋CRUD |
| 业主管理 | /api/owner/* | GET/POST/PUT/DELETE | 业主CRUD |
| 人房绑定 | /api/owner/room/* | GET/POST/DELETE | 绑定管理 |

详细接口文档见中期检查报告。

## 4. 安全设计

### 4.1 权限控制

| 角色 | 系统管理 | 社区管理 | 业主管理 | 数据统计 |
|------|---------|---------|---------|---------|
| 管理员(admin) | ✅ 全部 | ✅ 全部 | ✅ 全部 | ✅ 查看 |
| 普通用户(user) | ❌ 无权限 | ✅ 仅查看 | ✅ 仅查看 | ✅ 查看 |

### 4.2 admin账号保护

- admin账号不能被其他管理员修改基本信息
- admin账号不能被删除
- 通过请求头 `X-Username` 判断当前操作用户
